根据当前暂存区的变更，生成一条 git commit 提交信息，确认后执行提交。

## 执行步骤

1. 运行以下命令，了解本次变更的完整内容：
   - `git diff --staged` — 查看暂存区的具体变更
   - `git status` — 查看暂存文件列表

2. 根据变更内容，按照 **Conventional Commits** 规范生成提交信息：
   - 第一行格式：`<type>(<scope>): <description>`
     - `type` 从以下选择：`feat`（新功能）、`fix`（修复）、`docs`（文档）、`style`（格式）、`refactor`（重构）、`test`（测试）、`chore`（构建/工具/依赖等杂项）
     - `scope` 可选，填写受影响的模块或文件范围
     - `description` 使用**中文**简短祈使句描述变更，不超过 70 字符，不加句号
   - 若有重大破坏性变更，在 body 或 footer 中注明 `BREAKING CHANGE: <说明>`
   - 若变更较复杂，在标题后空一行，用 1~3 句中文补充说明原因或关键细节

3. 将最终提交信息展示给用户，**使用 AskUserQuestion 工具以选项方式询问用户**，提供以下选项：
   - ✅ 确认提交
   - ✏️ 修改提交信息
   - ❌ 取消提交
   - 若选择"修改提交信息"：询问用户希望如何修改，调整后重新展示并再次询问
   - 若选择"取消提交"：终止流程，提示用户已取消

4. 用户确认后直接执行 `git commit -m "..."` 完成提交，不要追加额外信息（如 Co-Authored-By）。

## 注意事项

- 如果暂存区为空，提示用户"暂存区没有变更，请先 git add"，不生成提交信息。
- 不要使用 `--no-verify` 跳过 hook。
