# 家庭影院系统产品文档 - Web 端

## 1. 文档目标与范围

### 1.1 目标
以页面/视图为主组织 Web 端需求，明确各页面的信息架构、交互行为与状态反馈。

### 1.2 范围
- 初始化引导
- 登录与登出
- 媒体库浏览与播放
- 上传与刷新操作
- 用户管理（管理员）
- 系统运维（管理员）

---

## 2. 用户角色与视角

| 角色 | 能力 |
|------|------|
| 管理员 | 全量媒体库访问 + 用户管理 + 系统运维 |
| 普通用户 | 仅访问白名单内的媒体库 |

---

## 3. 页面信息架构

```
/init                   首次初始化（仅系统无用户时可访问）
/login                  登录
/                       首页（媒体库列表）
/library/:id            媒体库详情（目录浏览）
/library/:id/play/:fid  视频播放
/admin
  /users                用户管理
  /system               系统运维（仪表盘 + 任务中心 + 日志）
```

---

## 4. 各页面详情

### 4.1 初始化页 `/init`

**访问规则**
- 仅当系统内尚无任何用户时可访问；
- 已有用户时访问此页面，自动跳转到 `/login`。

**页面内容**
- 表单：管理员用户名、密码、确认密码。
- 提交：创建首个管理员账号，成功后跳转到 `/login`。

**状态反馈**
- 用户名已存在 → 提示"用户名已被使用"（理论上首次初始化不会触发，但接口可能返回）。
- 密码不一致 → 前端本地校验，提示"两次密码输入不一致"。
- 提交失败 → 提示具体错误。

---

### 4.2 登录页 `/login`

**页面内容**
- 表单：用户名、密码。
- 提交：登录，成功后跳转到 `/`。

**状态反馈**
- 账号或密码错误 → 提示"用户名或密码错误"（不区分是哪个错误，避免信息泄露）。
- 账号已被禁用 → 提示"账号已被停用，请联系管理员"。
- Token 过期或被踢出（后续请求 401）→ 自动跳转到 `/login`，根据响应体 `error_code` 字段显示不同提示：
  - `token_expired` → "登录已过期，请重新登录"
  - `user_disabled` → "账号已被停用，请联系管理员"
  - `user_deleted` → "账号不存在，请联系管理员"
  - 其他/未知 → "登录已失效，请重新登录"

---

### 4.3 首页 `/`（媒体库列表）

**页面内容**
- 展示当前用户有权访问的所有媒体库列表（管理员看到全部）。
- 每张媒体库卡片展示：名称、类型（视频/相机）。
- 管理员额外操作：新建媒体库、删除媒体库。

**新建媒体库（管理员）**
- 弹窗表单：媒体库名称、目录路径、类型（视频/相机，单选）。
- 提交后自动触发初始全量刷新，页面上该媒体库卡片进入刷新状态（与详情页一致，每 3~5 秒轮询任务状态）；刷新完成后卡片状态自动恢复。
- 名称重复 → 提示"名称已存在，请修改后重试"。

**删除媒体库（管理员）**
- 二次确认弹窗，明确提示"删除后索引数据将被清理，用户播放记录不受影响"。
- 删除后该媒体库从列表移除；若有进行中的队列任务，提示"任务已中止"。

**空状态**
- 普通用户无可访问媒体库时：提示"暂无可访问的媒体库，请联系管理员开通权限"。
- 管理员无媒体库时：提示"还没有媒体库，立即创建"。

---

### 4.4 媒体库详情页 `/library/:id`

**页面内容**
- 页面头部：媒体库名称、类型、当前路径面包屑、操作按钮（手动刷新、上传文件）；管理员额外显示"改名"按钮。
- 改名（管理员）：点击后弹窗输入新名称，提交后实时更新页面标题；名称重复 → 提示"名称已存在，请修改后重试"。
- 目录与文件列表：展示当前路径层级下的子目录和文件。
  - 子目录：显示目录名，点击后进入该子目录（更新面包屑，刷新列表）。
  - 文件（视频类型）：仅展示 mp4 文件，显示文件名、时长、播放进度（已看 / 进度百分比）。
  - 文件（相机类型）：展示 mp4、jpg、png 文件；图片提供缩略图预览，视频显示文件名、时长、播放进度（已看 / 进度百分比）。
- 面包屑导航：展示从根目录到当前目录的路径，点击任意层级可快速跳回。
- 点击视频文件 → 跳转播放页。
- 点击图片文件 → 原图预览（lightbox 效果）。
- 支持关键字搜索：输入关键字后跨目录全局匹配文件名，结果展示匹配文件的完整路径。

**刷新状态**
- 队列有任务时，页面顶部显示刷新状态条：
  - "刷新中…"（有任务执行中）
  - "等待刷新…"（仅有待执行任务）
- 刷新完成后文件列表自动更新（轮询，每 3~5 秒请求一次任务状态）。

**手动刷新**
- 点击"刷新"按钮，后端推入全量刷新任务。
- 若队列中已有全量刷新任务（待执行或执行中），提示"刷新任务已在队列中，无需重复提交"（后端去重，前端可据此反馈）。
- 无权限时按钮不展示（普通用户不在白名单时）。

**上传文件**
- 点击"上传"按钮，打开文件选择器。
  - 视频类型库：仅接受 `.mp4`。
  - 相机类型库：接受 `.mp4`, `.jpg`, `.png`。
- 同一用户同一媒体库同时只能有 1 个上传任务；若已有上传中，后端返回 `409`，提示"当前有上传任务进行中，请等待完成后再上传"。
- 上传进度条实时展示；上传成功后自动触发定向刷新并更新文件列表。
- 上传中断（网络断开等）→ 标记失败，提示"上传失败，请重试"，提供重试按钮；重试为完整重新上传，不支持断点续传。
- 权限说明：普通用户不在白名单时，上传按钮不展示；若前端判断失误或权限在上传途中被撤销，后端返回 `403`，提示"您无权向此媒体库上传文件"。

**空状态**
- 媒体库已建但索引为空（初始刷新中）：展示"正在初始化，请稍候…"。
- 刷新完成后当前目录无内容：展示"此目录暂无文件"。

---

### 4.5 视频播放页 `/library/:id/play/:fid`

**页面内容**
- 视频播放器，支持：
  - 从上次进度续播（若有进度记录且未达 90%）。
  - 进度条拖拽、全屏、音量控制。
- 退出播放（离开页面）时上报进度（监听 `beforeunload` / `visibilitychange`，尽力而为）。
- 进度达到 90% 时，后端自动标记为已看完（前端可据此在文件列表展示"已看"标记）。

**状态反馈**
- 文件不存在（后端返回 404）→ 提示"文件不存在或已移除"，返回媒体库页。
- 播放失败（其他错误）→ 提示"播放失败，请稍后再试"。

---

### 4.6 用户管理页 `/admin/users`（仅管理员可见）

**页面内容**
- 用户列表：仅展示未删除用户（含已禁用）；用户名、角色、状态（正常/已禁用）、创建时间。
- 操作列：禁用/解禁、重置密码、删除。
- 顶部：新建用户按钮。

**新建用户**
- 弹窗表单：用户名、初始密码。
- 新建成功，用户默认无任何媒体库权限。

**媒体库权限配置**
- 点击用户行 → 展开或进入详情，展示媒体库权限配置（白名单列表，可勾选/取消）。
- 权限变更实时生效，静默保存，无需二次确认。

**禁用 / 解禁用户**
- 禁用时：弹出确认，提示"禁用后该用户将无法登录且当前会话立即失效"。
- 解禁时：直接执行，无需确认。
- 管理员自己的禁用按钮置灰（不可操作）。

**重置密码**
- 弹窗输入新密码，提交后该用户已有登录态立即失效。
- 管理员可修改自己和普通用户的密码。

**删除用户**
- 二次确认，提示"删除后该用户的历史行为数据将一并清除"。
- 管理员自己的删除按钮置灰（不可操作）。

**角色规则**
- 不展示"角色修改"入口（角色固定，不支持晋升/降级）。

---

### 4.7 系统运维页 `/admin/system`（仅管理员可见）

页面分为三个 Tab：仪表盘、任务中心、日志。

#### 4.7.1 仪表盘 Tab
- 媒体总数（全量）。
- 用户登录状态：展示各用户最近活跃时间（依赖 Token 最后使用时间），不做实时在线判断。

#### 4.7.2 任务中心 Tab
- 展示各媒体库的刷新队列状态。
- 每个媒体库：当前执行中任务、待执行任务列表（全量刷新 / 定向刷新）、最近完成任务。
- 任务状态标签：执行中 / 等待中 / 成功 / 失败。
- 失败任务提示已写入索引的部分结果，不影响后续任务。

#### 4.7.3 日志 Tab
- 统一展示所有日志（审计操作、刷新任务、上传、播放失败），每条记录显示时间和描述信息。
- 支持关键字筛选；分页展示，默认每页 30 条。

## 5. 全局交互规范

### 5.1 鉴权与跳转
- 所有需登录的页面，若未携带有效 Token，自动跳转到 `/login`。
- `/init` 在已有用户时自动跳转到 `/login`。
- 普通用户访问 `/admin/*` 页面 → 跳转到 `/` 并提示"无权访问"。

### 5.2 登出
- 全局导航栏右上角提供用户菜单，包含"登出"入口。
- 点击登出 → 调用登出接口，清除本地 Token，跳转到 `/login`。

### 5.3 权限变更的实时感知
- 用户被移出某媒体库白名单时，若当前正在浏览该媒体库，下一个操作请求（刷新/上传/播放）返回 403，前端提示"您已无权访问此媒体库"并跳转到 `/`。
- 用户被禁用时，下一个请求返回 401（`error_code: user_disabled`），跳转到 `/login`，提示"账号已被停用"。

### 5.4 加载与错误状态
- 列表/数据加载中展示骨架屏或 Loading 指示器。
- 接口异常（5xx）→ 页面级或操作级提示"服务异常，请稍后再试"。
- 网络断开 → 全局提示"网络已断开"。

### 5.5 响应式与设备
- 优先 PC 浏览器，移动端做基础适配（可访问、可播放，非深度优化）。

## 6. 前后端约定项

以下为前后端接口层面的关键约定，需在接口设计阶段落实：

1. **视频时长**：后端索引 mp4 文件时需读取并存储时长（duration），供文件列表展示。
2. **图片缩略图接口**：相机类型媒体库需提供缩略图接口（实时生成 + 磁盘缓存，约 400px 宽），原图通过图片浏览接口单独访问。
3. **手动刷新返回字段**：刷新接口返回 `{ "queued": true/false }` 区分"新推入"与"已去重"。
4. **上传错误码区分**：无权限返回 `403`，并发冲突返回 `409`。
5. **手动刷新权限拒绝**：无权限返回 `403`，与上传保持一致。
6. **401 响应体 error_code**：所有 401 响应体需包含 `error_code` 字段，枚举值：`token_expired` / `user_disabled` / `user_deleted`。
7. **文件列表播放进度**：文件列表接口需为每个文件返回当前用户的播放进度（进度百分比）与是否已看完标记，供文件列表展示播放进度与"已看"标记。
8. **上传重试策略**：上传不支持断点续传，重试为完整重新上传。
