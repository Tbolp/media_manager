# 家庭影院系统产品文档 - 移动端

## 1. 文档目标与范围

### 1.1 目标
以页面/视图为主组织移动端需求，明确各页面的信息架构、触控交互与状态反馈。移动端聚焦媒体库内容的展示与播放，同时支持手动刷新，不涉及文件上传与用户管理等后台管理操作。

### 1.2 范围
- 服务器地址配置
- 登录与登出
- 媒体库列表浏览
- 媒体库详情（目录浏览、搜索、文件展示）
- 视频播放（含续播）
- 图片浏览（全屏预览、左右滑动翻页）
- 手动刷新媒体库

### 1.3 不在范围内
- 系统初始化（移动端不支持首次初始化，引导用户使用 Web 端）
- 文件上传
- 用户管理
- 系统运维（仪表盘、任务中心、日志）
- 创建/删除/改名媒体库

---

## 2. 用户角色与视角

| 角色 | 移动端能力 |
|------|------------|
| 管理员 | 与普通用户相同——浏览、播放、刷新，用户管理等后台功能通过 Web 端使用 |
| 普通用户 | 浏览白名单内的媒体库，观看视频、查看图片、手动刷新 |

---

## 3. 页面信息架构

```
/server                     服务器地址配置（首次启动或未配置时）
/login                      登录
/                           首页（媒体库列表）
/library/:id                媒体库详情（目录浏览）
/library/:id/play/:fid      视频播放
```

> 移动端无 `/init` 和 `/admin/*` 路由；访问不存在的路由一律跳转到 `/`（已登录）或 `/login`（未登录）或 `/server`（未配置服务器地址）。
>
> App 启动时跳转优先级：未配置服务器地址 → `/server`；已配置但未登录 → `/login`；已登录 → `/`。

---

## 4. 全局导航与布局

### 4.1 导航模式
- 采用**顶部导航栏 + 页面内返回**的导航方式，无底部 Tab 栏（功能模块单一，不需要）。
- 顶部导航栏固定展示：
  - 左侧：返回箭头（非首页时展示）或应用 Logo（首页）。
  - 中间：当前页面标题（媒体库名、当前目录名等）。
  - 右侧：用户头像/昵称首字母，点击弹出菜单（含"登出"）。

### 4.2 安全区域
- 适配 iOS 刘海屏与 Android 异形屏的安全区域，使用 Flutter `SafeArea` widget 包裹页面内容。
- 视频播放页全屏时，控制条需避开安全区域。

## 5. 各页面详情

### 5.1 服务器配置页 `/server`

**触发时机**
- App 首次安装启动，尚未保存过服务器地址时，自动进入此页。
- 服务器地址有误或无法访问时（如服务器换了 IP），用户可通过登录页的"修改"入口跳回此页。

**页面内容**
- 输入框：服务器地址（示例占位文字：`http://192.168.1.100:8080` 或 `https://media.home.com`）。
- "保存并继续"按钮：保存地址到本地存储，跳转到 `/login`。

**地址格式校验**
- 前端基础校验：必须以 `http://` 或 `https://` 开头，不允许末尾带 `/`（自动 trim）。
- 不合法格式 → 输入框下方提示"请输入有效的服务器地址"，禁止提交。

**已有配置时的行为**
- 进入此页时，输入框预填当前已保存的地址，方便用户在原地址基础上修改。
- 保存新地址后，清除本地 Token（原服务器的登录态不再适用），跳转到 `/login`。

---

### 5.2 登录页 `/login`

**页面内容**
- 居中卡片：用户名输入框、密码输入框（支持密码显示/隐藏切换）、登录按钮。
- 使用移动端友好的键盘类型：用户名使用普通文本键盘，密码框开启 `obscureText`。
- 点击登录按钮后，按钮进入 Loading 状态，防止重复提交。

**系统初始化引导**
- 若后端返回"系统未初始化"状态，不跳转初始化页，而是展示提示：
  > "系统尚未完成初始化，请先在电脑端（Web）完成管理员账号创建。"

**状态反馈**
- 账号或密码错误 → 提示"用户名或密码错误"。
- 账号已被禁用 → 提示"账号已被停用，请联系管理员"。
- Token 过期或被踢出（后续请求 401）→ 自动跳转到登录页，根据 `error_code` 显示提示：
  - `token_expired` → "登录已过期，请重新登录"
  - `user_disabled` → "账号已被停用，请联系管理员"
  - `user_deleted` → "账号不存在，请联系管理员"
  - 其他 → "登录已失效，请重新登录"

**服务器地址入口**
- 登录页底部展示当前已配置的服务器地址（灰色小字，如 `http://192.168.1.100:8080`），旁边有"修改"链接，点击跳转到 `/server`。

---

### 5.3 首页 `/`（媒体库列表）

**页面内容**
- 展示当前用户有权访问的所有媒体库，以**竖向卡片列表**呈现（每行一张）。
- 每张卡片结构：
  - 上方：媒体库封面缩略图（使用 `GET /api/libraries` 返回的 `cover_file_id`，拼接 `GET /api/files/{cover_file_id}/thumbnail?token=<jwt>` 获取；`cover_file_id` 为空或缩略图未就绪时，用类型占位图标填充）。
  - 下方信息栏：左侧显示媒体库名称与类型图标，右侧显示刷新图标按钮。
- 卡片点击（非刷新图标区域）后进入对应媒体库详情页。
- 下拉刷新：支持下拉手势触发列表数据刷新（重新拉取媒体库列表）。

**刷新操作（卡片内）**
- 点击卡片右侧刷新图标，向后端推入该媒体库的全量刷新任务，图标进入转圈 Loading 状态；接口返回后立即恢复图标状态，后续刷新进度通过进入详情页后的顶部横幅感知。
- 若队列中已有全量刷新任务（待执行或执行中），提示"刷新任务已在队列中，无需重复提交"。

**空状态**
- 普通用户无可访问媒体库：展示图示 + "暂无可访问的媒体库，请联系管理员开通权限"。
- 管理员无媒体库：展示图示 + "还没有媒体库，请在电脑端创建"。

---

### 5.4 媒体库详情页 `/library/:id`

**页面头部**
- 顶部导航栏显示媒体库名称（根目录）或当前目录名（子目录）。
- 内容区顶部展示面包屑导航：从根目录到当前目录的完整路径，可横向滚动，点击任意层级跳回对应目录。

**目录与文件列表**
- 列表项分两类：**子目录** 和 **文件**，目录置顶，文件在下。
- 子目录：展示文件夹图标 + 目录名，点击进入子目录（更新面包屑与列表）。
- 文件（视频类型库）：
  - 展示视频缩略图（`GET /api/files/{fid}/thumbnail?token=<jwt>`，懒加载）、文件名、时长。
  - 若有播放进度：展示进度条（细长条于卡片底部）或"已看完"标记。
  - 点击进入视频播放页。
- 文件（相机类型库）：
  - **图片文件**（jpg/png）：展示缩略图（`GET /api/files/{fid}/thumbnail?token=<jwt>`，懒加载）、文件名；点击进入图片全屏预览。
  - **视频文件**（mp4）：同视频类型库逻辑。
- 列表滚动：采用虚拟列表或分页加载，防止大目录时渲染卡顿。

**相机库图片网格模式**
- 相机类型库默认使用**网格视图**（每行 3 张缩略图）；视频文件在网格中以封面+播放图标形式展示。
- 支持列表/网格视图切换按钮（右上角图标），用户切换后本次会话内保持选择，不随目录切换重置。

**关键字搜索**
- 顶部搜索栏（点击后展开）：输入关键字后跨目录全局匹配文件名，结果以列表展示，每条结果显示文件名及其完整路径。
- 搜索结果中的文件，点击行为与正常文件一致（视频 → 播放，图片 → 预览）。
- 清空搜索内容后，恢复正常目录浏览状态。

**刷新状态感知**
- 若媒体库当前有刷新任务进行中（轮询任务状态，每 5 秒一次），在页面顶部展示一条提示横幅："正在刷新索引，文件列表可能未完整"。
- 刷新完成后，横幅消失，自动拉取最新文件列表。

**空状态**
- 初始刷新中（索引为空）：展示"正在初始化，请稍候…"。
- 当前目录无文件：展示"此目录暂无文件"。

### 5.5 视频播放页 `/library/:id/play/:fid`

**播放器**
- 使用全屏沉浸式播放器，页面背景黑色。
- 默认横屏播放，设备竖屏时自动提示"旋转手机获得更好体验"（提示可关闭）；强制横屏锁定策略依系统能力而定，不强制锁定。
- 支持：
  - 续播：若有进度记录且未达 90%，从上次位置继续播放；首次播放或已看完则从头开始。
  - 进度条拖拽（触摸滑动）。
  - 左右横滑屏幕：自由拖动进度，滑动时展示目标时间点预览。
  - 双击左/右侧区域：快退/快进 15 秒。
  - 长按屏幕：2x 加速播放，松开恢复正常速度，加速中展示速度标识。
  - 播放/暂停（点击屏幕中央）。
  - 亮度调节：在屏幕左侧上下滑动调节亮度（尽力而为，依系统权限）。
  - 音量手势：在屏幕右侧上下滑动调节音量。

**进度上报**
- 退出播放页时自动上报当前进度，触发时机：
  - 点击返回键/手势返回（离开播放页）；
  - 切到后台或锁屏：通过 `WidgetsBindingObserver` 监听 `AppLifecycleState.paused` / `detached` 触发，尽力而为。
- 进度达到 90% 时，标记为已看完。

**状态反馈**
- 文件不存在（404）→ 提示"文件不存在或已移除"，返回媒体库详情页。
- 播放失败（其他错误）→ 提示"播放失败，请稍后再试"，提供重试按钮。
- 网络中断导致播放卡顿 → 播放器原生 Buffering 状态展示，自动恢复。

---

### 5.6 图片预览（相机类型库）

**触发方式**
- 在媒体库详情页点击图片文件，以全屏覆盖层形式展示（非独立路由，覆盖在详情页之上）。

**预览交互**
- 全屏展示原图（懒加载，加载中展示缩略图占位）。
- 左右滑动翻页：在同一目录（或搜索结果）内切换上一张/下一张图片；视频文件在预览序列中跳过（不在图片翻页序列内）。
- 双指捏合/展开缩放图片；双击快速放大/恢复原始大小。
- 单击屏幕中央切换显示/隐藏顶部信息栏（文件名、当前第几张/共几张）。
- 顶部展示"关闭"按钮（或向下滑动手势关闭），返回媒体库详情页。

**状态反馈**
- 图片加载失败 → 展示加载失败占位图 + "图片加载失败"文字。

## 6. 全局交互规范

### 6.1 鉴权与跳转
- 所有需登录的页面，若未携带有效 Token，跳转到 `/login`。
- `/login` 页面在已登录状态下自动跳转到 `/`。

### 6.2 登出
- 点击右上角用户菜单 → 选择"退出登录" → 调用登出接口，清除本地 Token，跳转到 `/login`。

### 6.3 权限变更的实时感知
- 用户被移出某媒体库白名单：下一次请求返回 `403`，提示"您已无权访问此媒体库"，返回首页。
- 用户被禁用：下一次请求返回 `401 user_disabled`，跳转到 `/login`，提示"账号已被停用"。

### 6.4 加载与错误状态
- 列表加载中展示骨架屏。
- 图片缩略图懒加载，加载中展示灰色占位块。
- 接口异常（5xx）→ Toast 提示"服务异常，请稍后再试"。
- 网络断开 → 全局顶部横幅提示"网络已断开"，恢复后自动隐藏。

### 6.5 触控体验规范
- 所有可点击区域最小触控尺寸不低于 44×44pt（iOS HIG 标准）。
- 列表项点击有高亮反馈（`active` 态）。
- 操作按钮（如关闭、返回）避免置于屏幕边缘触控盲区。

### 6.6 网络与缓存
- 图片缩略图使用 HTTP 缓存（依赖后端 `Cache-Control` 响应头），避免重复加载。
- 视频播放不做本地缓存，依赖后端 HTTP Range 支持实现流式播放。

## 7. 技术选型方向

> 本节为方向性建议，具体方案待技术设计阶段确认。

| 方向 | 说明 |
|------|------|
| 跨平台方案 | Flutter，同一套代码构建 iOS 与 Android，与现有后端 API 对接 |
| 视频播放 | 使用 `video_player` + `chewie` 组件，或 `better_player`，保证 MP4 流畅播放与手势控制 |
| 图片预览 | 使用 `photo_view` 组件，支持手势缩放、双击放大与懒加载 |
| 网络层 | 使用 `dio` 作为 HTTP 客户端，封装请求拦截器（Token 注入、401 处理、服务器地址动态切换） |
| 状态管理 | 使用 `riverpod` 或 `bloc` 管理全局状态；登录态持久化到 `flutter_secure_storage`（底层对应 Keychain / Keystore） |

## 8. 与 Web 端的差异对照

| 功能 | Web 端 | 移动端 |
|------|--------|--------|
| 服务器地址配置 | ❌ 不需要（域名固定） | ✅ 首次启动或随时修改 |
| 系统初始化 | ✅ 支持 | ❌ 不支持，引导用 Web |
| 媒体库列表浏览 | ✅ | ✅ |
| 目录浏览与搜索 | ✅ | ✅ |
| 视频播放（续播） | ✅ | ✅ |
| 图片预览 | ✅ Lightbox | ✅ 全屏 + 滑动翻页 |
| 横屏沉浸播放 | 有限支持 | ✅ 重点优化 |
| 文件上传 | ✅ | ❌ 此版本不支持 |
| 手动刷新媒体库 | ✅ 详情页操作 | ✅ 首页卡片操作 |
| 用户管理 | ✅（管理员） | ❌ 不支持 |
| 系统运维 | ✅（管理员） | ❌ 不支持 |
| 网格视图（相机库） | ❌ | ✅ 移动端新增 |
| 手势操作（音量/亮度/快进） | ❌ | ✅ 移动端新增 |
