# 家庭影院系统产品文档 - 后端

## 1. 文档目标与范围

### 1.1 目标
以模块为主组织后端需求，先确认模块边界与职责，再进入接口和实现阶段。

### 1.2 产品定位
本系统以媒体共享为核心目标，面向家庭或小团体场景。权限设计以信任为前提，不以管控用户行为为出发点；管理员的职责是管理资源访问，而非监控或限制用户的个人行为数据。

### 1.3 范围
- 用户与权限
- 媒体库与刷新上传任务
- 媒体播放
- 用户行为数据（进度、历史）
- 日志与运维支撑

### 1.4 非范围
- 本阶段不展开接口定义和实现细节
- 外部元数据抓取（TMDB/TVDB/豆瓣等）
- 推荐算法
- 在线流媒体聚合
- 转码

## 2. 模块划分与功能需求

### 2.1 用户模块
职责：账号体系与授权控制。
- 首次初始化：仅当系统内无任何用户时，初始化接口才可被调用，用于创建首个管理员用户。
- 账号基础规则：
  - 用户名全局唯一。
  - 密码仅存哈希（Argon2），不保存明文。
  - 认证方式：用户名 + 密码登录，登录后签发 Token，Token 有过期时间；后续请求携带 Token 鉴权，每次请求均实时校验用户当前状态，确保禁用、删除等变更立即生效；用户数据维护一个版本号，Token 签发时写入当前版本号，每次请求校验 Token 时比对版本号，不一致则拒绝；需要使登录态立即失效时，递增版本号即可使该用户所有已签发 Token 立即失效。
  - 所有 401 响应体需包含 `error_code` 字段，枚举值：`token_expired`（Token 过期或已失效）、`user_disabled`（用户已禁用）、`user_deleted`（用户已删除）。
- 账号能力边界：
  - 管理员：可创建、删除、禁用、解禁普通用户；可修改普通用户及自己的密码。
  - 管理员限制：不可禁用或删除自己。
  - 普通用户：无账号管理权限，不可修改自身用户名或密码。
- 角色：管理员、普通用户。角色固定，不支持晋升或降级；管理员唯一来源为系统首次初始化时创建的首个用户。
- 权限：
  - 可访问的媒体库白名单（存储在 users 表的 JSON 字段中，默认为空；管理员可见所有媒体库且不受白名单限制）。
- 权限生效规则：
  - 权限变更实时生效，当前会话的后续请求即刻受新权限约束。
  - 白名单内的用户即可对该媒体库进行上传与刷新；已在执行中的任务不因中途改权被中断或拦截。
- 关键规则补充：
  - 删除用户为软删除，保留用户基础信息（ID、用户名、删除时间）用于审计关联，用户名释放后可被新用户使用；关联的历史行为记录一并删除，已有登录态立即失效。
  - 用户被禁用后，已有登录态立即失效，后续登录请求被拒绝。
  - 用户可主动登出，登出后当前登录态立即失效。
  - 任意用户密码被修改后，该用户已有登录态立即失效。
  - 用户创建/删除/禁用/解禁/权限变更/密码修改须通过日志模块记录审计日志（操作者、目标用户、时间、变更项）。
  - 媒体库创建/删除/改名须通过日志模块记录审计日志（操作者、目标媒体库、时间、变更项）。
  - 新建用户默认无任何权限，由管理员按需开启。

### 2.2 用户行为模块
职责：沉淀用户行为数据，独立于用户模块。
- 播放进度（断点续播）：
  - 进度按媒体文件维度存储。
  - 退出时上报当前进度。
  - 播放进度达到 90% 标记为已看完，进度记录保留。
- 播放历史（最近观看）：
  - 开始播放即记录，同时更新该条记录的时间戳。
  - 同一媒体文件多次观看只保留最新一条。
  - 本版本不提供用户自行删除历史的能力。
- 数据按用户隔离；媒体库白名单仅控制访问入口，用户被移出白名单不影响已有的行为数据（进度、历史）；用户被禁用后行为数据保留；用户被删除后行为数据一并清除。播放进度与历史仅适用于视频文件，图片文件不记录行为数据。

### 2.3 媒体库模块
职责：媒体库生命周期、索引任务管理与媒体数据查询。

库管理：
- 媒体库以目录为单位创建，索引该目录下的文件内容；创建时需提供名称与目录路径，名称全局唯一，用于展示与标识。
- 媒体库的创建、删除、改名仅限管理员操作。
- 媒体库类型：
  - 视频类型：将目录中的 mp4 文件索引进库。
  - 相机类型：将目录中的 mp4、jpg、png 文件索引进库。
- 删除媒体库：清理对应索引数据，用户的播放进度与历史数据不受影响；删除操作直接执行，不经过队列，队列中待执行或执行中的任务立即中止。
- 媒体库路径与类型不可变更，名称可修改且修改后仍须全局唯一。

文件管理：
- 库内文件不支持删除。
- 用户可上传文件至媒体库（需在该媒体库白名单内）；上传成功后自动触发索引更新。
- 用户可触发手动刷新（需在该媒体库白名单内）；无权限返回 `403`。

索引与刷新：
- 以目录结构为标准，索引保留文件相对于媒体库根目录的完整路径（如 `a/b/c.mp4`），用于还原目录层级。
- 新建媒体库时，自动向刷新队列提交一次初始化全量刷新任务。
- 不监听本地存储变化，仅在手动刷新或上传后同步索引。
- 文件识别策略：
  - 视频类型：递归扫描目录，索引所有 mp4 文件，保留各文件的相对路径。
  - 相机类型：递归扫描目录，索引所有 mp4、jpg、png 文件，保留各文件的相对路径。

上传策略：
- 上传直接执行，不经过队列；单用户同一媒体库最多同时进行 1 个上传任务，并发时返回 `409`。
- 上传成功后，自动向刷新队列推送一条针对该文件的定向刷新任务以更新索引。
- 上传中断标记失败，支持用户手动重试；重试为完整重新上传，不支持断点续传。
- 用户在媒体库白名单内方可上传，无权限直接拒绝；视频类型库仅接受 mp4 文件，相机类型库接受 mp4、jpg、png 文件。
- 上传拒绝错误码约定：无权限（不在白名单）返回 `403`；并发冲突（已有上传任务进行中）返回 `409`。

刷新队列策略：
- 队列粒度：每个媒体库一个独立串行队列，所有索引写入均通过刷新队列执行。
- 任务类型：全量刷新（覆盖初始化和手动刷新场景）、定向刷新（上传成功后针对该文件触发）。
- 刷新合并：已有待执行/执行中的全量刷新时，后续全量刷新请求去重合并；定向刷新不合并，按序执行；队列中的任务严格按序执行，全量刷新不可跳过前置的定向刷新任务。
- 刷新失败：已扫描的部分结果写入索引，记录日志，不阻塞后续任务。
- 手动刷新接口返回约定：返回 `{ "queued": true }` 表示任务已新推入队列；返回 `{ "queued": false }` 表示队列中已存在全量刷新任务，本次请求被去重合并，无需重复执行。

媒体数据与查询：
- 索引存储文件相对于媒体库根目录的完整路径，供目录层级还原。
- 提供目录浏览接口：传入当前路径前缀，返回该层级下的子目录列表与文件列表；根路径返回根层级的子目录和文件。
- 支持按文件名关键字全局检索（跨目录匹配）。
- 视频文件索引时需读取并存储时长（duration），供前端文件列表展示。

### 2.4 媒体库播放模块
职责：围绕媒体库资源提供播放与浏览能力。
- 视频直传：源文件直接返回文件流，支持 HTTP Range。
- 图片浏览：相机类型媒体库中的图片文件提供直接访问能力（原图）。
- 图片缩略图：相机类型媒体库需提供缩略图接口，按需生成（约 400px 宽）并缓存至磁盘，后续请求直接返回缓存；供前端文件列表展示使用，点击后通过原图接口访问。
- 播放无需额外权限，用户在媒体库白名单内即可播放/浏览。

### 2.5 日志模块
职责：统一日志记录与查询。
- 所有日志（审计操作、刷新任务、上传、播放失败）统一写入单张日志表，每条记录仅包含时间和描述信息。
- 提供关键字筛选查询能力。

### 2.6 系统运维模块
职责：运行态观测与基础运维能力。
- 仪表盘指标：媒体总数、用户登录状态（各用户最近 Token 活跃时间，不做实时在线判断）；每次请求鉴权通过时更新该用户的最近活跃时间戳并持久化。
- 任务中心：展示各媒体库队列中的全量刷新与定向刷新任务状态。
- 系统稳定性保障：异常可观测。
